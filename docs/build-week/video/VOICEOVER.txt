GuardianNode — Guardian Review | target 2 minutes 48 seconds

[0:00–0:20]
A worrying message can be serious, a joke, or missing context. I built
GuardianNode to help parents protect a child without turning an uncertain signal
into an accusation. Detection stays local first; the parent decides what leaves
the system and what happens next.

[0:20–0:40]
Before Build Week, my GuardianNode platform already had a visible Windows agent,
a parent-owned backend, local rules, optional Ollama models, encrypted evidence,
authentication, a dashboard, and installers. Claude assisted with portions of
the earlier web interface. Guardian Review is the Build Week extension I created
with Codex and GPT-5.6.

[0:40–1:05]
This is a real client-server run using only synthetic data. A Windows child
agent captured a mild fictional gaming chat and uploaded encrypted evidence to
the isolated parent server. Local OCR and deterministic rules flagged the short
phrase as a medium bullying signal. The alert, screenshot, rule, score, and audit
trail are visible in the normal dashboard.

[1:05–1:45]
Guardian Review loads the authorized incident from the server rather than
trusting a browser payload. My deterministic minimizer removes names, accounts,
contact details, paths, device identifiers, precise locations, and unrelated
context. The parent sees the exact outbound JSON, can remove optional fields,
and must explicitly approve external processing. In this recording, the live
server-side OpenAI Responses API calls GPT-5.6 with no tools, store false, and a
strict versioned schema. The API key never enters the browser or repository.

[1:45–2:10]
GPT-5.6 returns a structured, locally persisted second opinion. It separates
observation from inference, surfaces benign explanations and missing context,
and recommends a calm, curious opening. The interface says not to punish or
accuse a child based only on AI. Immediate safety stays separate from later
discipline, and parent feedback is stored locally with the assessment version.

[2:10–2:35]
I used Codex with GPT-5.6 to preserve the pre-Build-Week baseline, trace the real
agent-to-dashboard path, implement the backend, strict schema, redaction,
consent, persistence, UI, failure handling, and tests. Codex also helped build
fifty-five synthetic evaluation cases and adversarial checks. That work changed
my design: runtime review stays on the tool-free Responses API instead of a
coding-agent transport with local tools.

[2:35–2:48]
GuardianNode keeps the parent in control. Guardian Review is an alpha second
opinion—not a diagnosis, emergency service, truth detector, punishment engine,
or replacement for a trusted family conversation.
