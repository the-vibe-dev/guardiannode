## Security/adversarial review

The review covered authentication, authorization/IDOR, API-key handling, prompt
injection, stored XSS, log leakage, sensitive-data exposure, CSRF, rate limits,
resource exhaustion, oversized evidence, and dependency vulnerabilities.

Two Guardian Review findings were fixed:

- The coding-agent provider could expose untrusted evidence to local read tools
  and inherited credentials. Provider selection and login now fail closed until
  enforceable zero-tool isolation exists.
- Deterministic minimization missed bracket-dot email forms, IPv6, and complete
  Windows path suffixes. Redaction v3 closes these cases and invalidates stale
  consent previews.

Additional release hardening removes setup-token prefixes from logs, enforces
positive list limits, caps/concurrency-gates local text-model input, and prevents
demo reset from deleting prefix-collision records.
